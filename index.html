Sentiment Analysis

Snippet of a project for a client

Project Description

For the course of this project, I applied and performed web, data scrapping on Google Reviews and TripAdvisor, along with customer sentiment analysis through R and Python.

Introduction

In this project for a client, I learnt firsthand on performing web scrapping as well as the application of Natural Language Processing. The aim of this project was to classify the Google Reviews of my client's business to have postive or negative connotations, and develop wordclouds so as our client could have a greater knowledge on the plus and pain points from their consumer's perspective.

All data was scrapped from public data of Google Reviews and TripAdvisor. Following this, I performed some cleaning to break all text down prior to analysation.

Pulling the data

As the aim was to perform Customer Sentiment Analysis, I manually scrapped data from TripAdvisor with the help of R Programming. Reviews from Google Reviews were pulled through the help of external source code from GitHub, which was consolidated into a single CSV file.

Sample Codes on Scrapping TripAdvisor


Following the consolidation of all reviews, I imported the CSV files obtained and converted them to dataframes to work on, using Python.

Importing libraries

Prior to our text analysis, I first imported all appropriate libraries for NLTK.

Data cleaning

A series of data cleaning methods were imposed, to allow for manageable datasets:
1. Stop Words
2. Punctuation Removal
3. Stemming, Lemmatization

For example, the output of my column 'Processed' which I created no longer has punctuations, and has kept only the important keywords that could potentially prove to value-add for the business.

WordCloud

Moving forward upon pre-processing of data, I performed a visualisation of the text data through WordCloud. For convenience sake, I stuck to using Python for this, despite it being equally doable in R.

This was done with the help of WordCloud library to create the object.

Sentiment Scoring

Finally, we looked at the main focus of this analysis, sentiment scoring. This was performed through the rule-based approach. Firstly, I used a pre-defined list of dictionaries that included positive/negative words, incrementers, decrementers and inverters. Then, I performed Part-of-Speech (POS) tagging, and dictionary-based scoring.

Scoring is performed in a manner where if a term was identified to be positive, it would be assigned a score of +1, whilst negative terms would be assigned -1 vice versa. Overall cumulated scoring aims to allow our business partner to come to a conclusion on their consumers approach towards their product and service.

Sample of assignment of pos+/neg+ connotations to text data.

Final Words

This project has allowed me to lay my foundations and gain a deeper understanding of how web scrapping works. It has helped me to also hone my analytical skillsets in both R and Python, and applying that towards an effective business use.

We went over several key NTKL methods that will prove to be fruitful when it comes to processing and cleaning of data. Futhermore, we leveraged on dictionaries for sentiment scoring.
